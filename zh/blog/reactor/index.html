简述Reactor原理

2026年1月13日

一、内容简介#

一句话概括Reactor模式 = I/O多路复用+I/O非阻塞+事件循环，不同的I/O事件，执行不同的回调函数（Callback Function）。本文将分析Reactor模式的原理以及实现，并且基于此分析其相比于其他方法在高并发下的性能优势。

二、Reactor的理论基础#

2.1 关于I/O多路复用（epoll）#

epoll的核心优势在于其高效的数据结构：使用红黑树来管理全部需要监控的描述符（简称为整fd集）、双向链表来管理已就绪的描述符（就绪fd集），并且在事件就绪的时候由内核直接通知用户空间，避免不必要的内存复制。支持水平触发LT和边沿触发ET，值得注意的是边沿触发ET：仅在fd状态发生变化的时候通知一次，避免了事件的重复触发，需要配合I/O非阻塞处理使用。

与epoll有关的函数：

 1//创建epoll实例，得到epfd
 2int epoll_create(int size)
 3
 4//控制事件（操作红黑树）
 5//参数op：EPOLL_CTL_ADD/MOD/DEL
 6int epoll_ctl(int epfd, int op, int fd, struct epoll_event *event)

2.2 事件#

1// epoll_event结构体
2struct epoll_event {
3    uint32_t events;    // 事件类型（EPOLLIN等，见下）
4    epoll_data_t data;  // 附加数据（用户数据）
5};

在网络编程中，事件主要是：

  1. 可读事件（EPOLLIN）：有数据可以读了
  2. 可写事件（EPOLLOUT）：可以发送数据了
  3. 错误事件（EPOLLERR）：出错了
  4. 挂起事件（EPOLLHUP）：连接断开了

3.3 主循环（mainloop）#

 1int main() {
 2
 3	unsigned short port = 2000;
 4
 5
 6	epfd = epoll_create(1);
 7
 8	int i = 0;
 9
10	for (i = 0;i < MAX_PORTS;i ++) {
11		
12		int sockfd = init_server(port + i);
13		
14		conn_list[sockfd].fd = sockfd;
15		conn_list[sockfd].r_action.recv_callback = accept_cb;
16		
17		set_event(sockfd, EPOLLIN, 1);
18	}
19
20	gettimeofday(&begin, NULL);
21
22	while (1) { // mainloop
23
24		struct epoll_event events[1024] = {0};
25		int nready = epoll_wait(epfd, events, 1024, -1);
26
27		int i = 0;
28		for (i = 0;i < nready;i ++) {
29
30			int connfd = events[i].data.fd;
31
32			if (events[i].events & EPOLLIN) {
33				conn_list[connfd].r_action.recv_callback(connfd);
34			} 
35
36			if (events[i].events & EPOLLOUT) {
37				conn_list[connfd].send_callback(connfd);
38			}
39
40		}
41
42	}
43	
44
45}

关键技术阐述：while(1)循环保证服务器不断工作，当有多个io事件发生时，epoll_wait将返回对应的就绪事件集（epoll批量返回）。然后就能通过调用不同的回调函数来处理了。

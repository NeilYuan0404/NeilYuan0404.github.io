Mysql实战——二进制数据读写

2025年11月25日

Mysql提供了许多C API函数，借此我们可以使用实现一个客户端（位于NS）用于对mysql进行增删查改，最终实现图片数据的读写访问功能

一、增删查改#

几个常见SQL语句

  • SELECT
  • INSERT
  • DELETE 这些可以通过#define宏定义的方式定义出来，避免每次都需要将SQL命令重新输入，也有更好的可读性。

增删查改的API函数 实现select：int neil_mysql_select(MYSQL *handle) 实现insert：int mysql_write(MYSQL *handle, char *buffer, int length)

值得注意的是，以上三个函数都拥有一个叫做handle的参数，名字叫做MYSQL连接句柄。有了它，我们才可以从NS连接到数据库，并且进行我们想要的操作，并且当操作不成功的时候，他可以通过mysql_error(handle)返回错误值。

1.1 select查询函数#

函数mysql_real_query（）可用于执行SQL语句，其返回值是int（只是为了方便错误处理），真正的内容传递依赖于MYSQL连接句柄，在mysql_real_query中我们想要的查询语句直接作为一个参数传入此函数即可，配合mysql_store_result(handle)，我们可以将服务器的内容存储到客户端的内存当中，其载体是MYSQL_RES，不过我们并不需要了解它的细节。

当我们希望按照行和列打印所返回内容的时候，使用mysql_fetch_row和mysql_fetch_fields，把MYSQl_RES这个结构体的实例作为参数传递进去即可。

 1int neil_mysql_select(MYSQL *handle) {
 2  if (mysql_real_query(handle, SQL_SELECT_TBL_USER, strlen(SQL_SELECT_TBL_USER))) {
 3    printf("mysql_real_query: %s\n", mysql_error(handle));
 4  }
 5
 6  MYSQL_RES *res = mysql_store_result(handle);
 7  if (res == NULL) {
 8    printf("mysql_real_query: %s\n", mysql_error(handle));
 9    return -2;
10  }
11
12  int rows = mysql_num_rows(res);
13  printf("rows: %d\n", rows);
14
15  int fields = mysql_num_fields(res);
16  printf("fields: %d\n",fields);
17
18  //取出并且显示结果
19  MYSQL_ROW row;
20  while ((row = mysql_fetch_row(res))) {
21
22    int i = 0;
23    for (i = 0; i < fields; i++) {
24      printf("%s\t", row[i]);
25    }
26    printf("\n");
27  }
28  
29  
30  mysql_free_result(res);
31
32  return 0;
33}

1.2 二进制数据写入和读取函数#

读和写的实现是镜像地，所以我们只要搞懂其中一个就能知道另一个该如何去编程。与之前select函数所使用的mysql_real_query不同，现在我们尝试采用预处理语句的方式。 预处理语句句柄：MYSQL_STMT，预处理语句好处：

  • 安全性：有效防止注入攻击
  • 高性能：重复的查询，亦只需要一次编译
  • 二进制数据：更好支持大二进制数据类型BLOB

使用的时候先使用mysql_stmt_init来初始化预处理语句句柄MYSQL_STMT，然后使用mysql_stmt_prepare将句柄关联到我们的目标sql语句，这里还有一个特别的步骤，使用mysql_stmt_bind_param进行参数绑定，这是为了提前告诉服务器所需要传递参数的类型。

MYSQL_BIND参数说明：

  • buffer_type = MYSQL_TYPE_LONG_BLOB - 指定参数为二进制大对象类型
  • buffer = NULL - 不直接提供数据缓冲区，使用分段发送
  • is_null = 0 - 明确表示该参数不是NULL值
  • length = NULL - 长度信息将在mysql_stmt_send_long_data中提供

之后的代码就是用mysql_stmt_execut执行预处理语句，即分段发送二进制数据，完成后关闭即可。

 1int mysql_write(MYSQL *handle, char *buffer, int length) {
 2
 3  if(handle == NULL || buffer == NULL || length <= 0) return -1;
 4  //statement
 5  MYSQL_STMT *stmt = mysql_stmt_init(handle);
 6  int ret = mysql_stmt_prepare(stmt, SQL_INSERT_IMG_USER, strlen(SQL_INSERT_IMG_USER));
 7  if (ret) {
 8    printf("mysql_stmt_prepar: %s\n", mysql_error(handle));
 9    return -2;
10  }
11
12  MYSQL_BIND param = {0};
13  param.buffer_type = MYSQL_TYPE_LONG_BLOB;
14  param.buffer = NULL;
15  param.is_null = 0;
16  param.length = NULL;
17
18  ret = mysql_stmt_bind_param(stmt, &param);
19  if (ret) {
20    printf("mysql_stmt_bind_param: %s\n", mysql_error(handle));
21    return -3;
22  }
23  //分段将数据发到服务器
24  ret = mysql_stmt_send_long_data(stmt, 0, buffer, length);
25    if (ret) {      
26      printf("mysql_stmt_send_long_data: %s\n", mysql_error(handle));
27      return -4;  
28    }
29  //执行：插入到服务器里
30  ret = mysql_stmt_execute(stmt);
31  if (ret) {
32    printf("mysql_stmt_execute: %s\n", mysql_error(handle));
33    return -5;  
34  }
35  //关闭
36  ret = mysql_stmt_close(stmt);
37  if (ret) {
38    printf("mysql_stmt_close: %s\n", mysql_error(handle));
39    return -6;  
40  }
41
42  return ret;
43}

二、图片读写#

我们的目标是提供一个接口，用于在主函数main当中调用，可以直接操作我们想要的图片文件。

2.1 图片写入#

 1int read_image(char *filename, char *buffer) {
 2  if (filename == NULL || buffer == NULL) return -1;
 3  
 4    FILE *fp = fopen(filename, "rb"); //fp指向文件的开头
 5    if (fp == NULL) {
 6      printf("fopen failed\n");
 7      return -2;
 8    }
 9
10    //file size
11    //下面这个函数将fp指向末尾
12    fseek(fp, 0, SEEK_END);
13    int length = ftell(fp);//偏移量
14    fseek(fp, 0, SEEK_SET);
15
16    int size = fread(buffer, 1, length, fp);
17    if (size != length) {
18      printf("fread failed:%d\n", size);
19      return -3;
20    }
21
22    fclose(fp);
23
24    return size;
25}

技术阐释：以上函数的作用就是从filename这个路径中获取我们想要的文件，并将其导入到buffer当中，然后，我们之前实现的二进制数据处理函数就可以顺利地从buffer当中找到需要发送到Mysql服务器的二进制图片数据。

你可能注意到了，从图片格式到二进制数据流这个转换的过程究竟是如何完成的？—答案是fopen，因为我们指定了一个fopen中的第二个参数为“rb”：

  • r代表read only，只读
  • b代表二进制

如果不指定b这个参数，则默认使用文本模式导入，一定要注意的是，图片格式一定不可以用文本模式导入，这会导致一些严重的格式问题，比如说换行符的转换。

文件模式说明：

  • "rb" - 二进制只读模式（图片读取）
  • "wb" - 二进制只写模式，会清空原文件
  • "wb+" - 二进制读写模式，会清空原文件 对于图片保存，使用"wb"更为合适

2.2 图片读取#

几乎和图片写入对称，不多赘述。

 1int write_image(char *filename, char *buffer, int length) {
 2  if (filename == NULL || buffer == NULL || length <= 0) return -1;
 3
 4  FILE *fp = fopen(filename, "wb+"); //fp指向文件的开头
 5  if (fp == NULL) {
 6    printf("fopen failed\n");
 7    return -2;
 8  }
 9
10  int size = fwrite(buffer, 1, length, fp);
11  if (size != length) {
12    printf("fwrite failed: %d\n", size);
13    return -3;
14  }
15
16  fclose(fp);
17  
18  return size;
19  
20}

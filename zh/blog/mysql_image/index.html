Mysql实战——图片读写

2024年11月28日

图片读写#

我们的目标是提供一个接口，用于在主函数main当中调用，可以直接操作我们想要的图片文件。

一、图片写入

 1int read_image(char *filename, char *buffer) {
 2  if (filename == NULL || buffer == NULL) return -1;
 3  
 4    FILE *fp = fopen(filename, "rb"); //fp指向文件的开头
 5    if (fp == NULL) {
 6      printf("fopen failed\n");
 7      return -2;
 8    }
 9
10    //file size
11    //下面这个函数将fp指向末尾
12    fseek(fp, 0, SEEK_END);
13    int length = ftell(fp);//偏移量
14    fseek(fp, 0, SEEK_SET);
15
16    int size = fread(buffer, 1, length, fp);
17    if (size != length) {
18      printf("fread failed:%d\n", size);
19      return -3;
20    }
21
22    fclose(fp);
23
24    return size;
25}

技术阐释：以上函数的作用就是从filename这个路径中获取我们想要的文件，并将其导入到buffer当中，然后，我们之前实现的二进制数据处理函数就可以顺利地从buffer当中找到需要发送到Mysql服务器的二进制图片数据。

你可能注意到了，从图片格式到二进制数据流这个转换的过程究竟是如何完成的？—答案是fopen，因为我们指定了一个fopen中的第二个参数为“rb”：

  • r代表read only，只读
  • b代表二进制

如果不指定b这个参数，则默认使用文本模式导入，一定要注意的是，图片格式一定不可以用文本模式导入，这会导致一些严重的格式问题，比如说换行符的转换。

文件模式说明：

  • "rb" - 二进制只读模式（图片读取）
  • "wb" - 二进制只写模式，会清空原文件
  • "wb+" - 二进制读写模式，会清空原文件 对于图片保存，使用"wb"更为合适

二、图片读取

几乎和图片写入对称，不多赘述。

 1int write_image(char *filename, char *buffer, int length) {
 2  if (filename == NULL || buffer == NULL || length <= 0) return -1;
 3
 4  FILE *fp = fopen(filename, "wb+"); //fp指向文件的开头
 5  if (fp == NULL) {
 6    printf("fopen failed\n");
 7    return -2;
 8  }
 9
10  int size = fwrite(buffer, 1, length, fp);
11  if (size != length) {
12    printf("fwrite failed: %d\n", size);
13    return -3;
14  }
15
16  fclose(fp);
17  
18  return size;
19  
20}
